# Sound Processor

CLI application for WAV audio processing implemented in C++20.

The project implements the required `sound_processor` and `sound_processor_tests` CMake targets. The application reads PCM mono 44100 Hz 16-bit WAV files, applies a pipeline of filters passed through command-line arguments, and writes the result to a WAV file.

## Build

From the repository root:

```bash
cmake -S . -B build
cmake --build build
```

The executable is created as:

```text
build/sound_processor
```

## Tests

Build and run unit tests with Catch2:

```bash
cmake -S . -B build
cmake --build build --target sound_processor_tests
ctest --test-dir build --output-on-failure
```

The tests are located in:

```text
sound_processor_1/tests/sound_processor_tests/
```

## CLI usage

```bash
build/sound_processor [-i input.wav] [-o output.wav] [-f filter args...] ...
```

Examples:

```bash
build/sound_processor -i input.wav -o output.wav -f ampl 0.8
build/sound_processor -o sine.wav -f generator sin 440 2000
build/sound_processor -i input.wav -o output.wav -f lowpass 51 -f timestretch 1.5
```

If no filters are provided, the input WAV is copied to the output WAV unchanged. If no output file is provided, the result is not written. If no input file is provided and the first filter is not a generator, an empty waveform is used.

## WAV requirements

Supported WAV format is fixed by the assignment:

```text
RIFF/WAVE
PCM
mono
44100 Hz
16-bit signed integer samples
```

The reader rejects files with unsupported format, channel count, sample rate, bit depth, invalid RIFF size, invalid `byte_rate`, invalid `block_align`, or truncated chunks.

## Filters

Transforming filters:

```text
ampl        <factor>
normalize   [peak]
silence     <sec|ms> <start> <end>
timestretch <factor>
lowpass     <window_size>
highpass    <window_size>
bandpass    <low_window_size> <high_window_size>
reject      <low_window_size> <high_window_size>
mute        <start_sec> <end_sec>
mix         <file.wav> <start_sec>
```

Generator filters ignore the input waveform and create a new one:

```text
generator sin <frequency_hz> <duration_ms>
generator am  <amplitude> <carrier_hz> <modulation_hz> <depth> <duration_ms>
generator fm  <amplitude> <carrier_hz> <modulation_hz> <deviation_hz> <duration_ms>
```

Parameter constraints follow the assignment: non-negative amplification, normalized peak in `[0, 1]`, positive time stretch factor, odd window sizes for frequency filters, valid generator amplitudes/depths, and non-negative durations.
